ArgoCD Hub-and-Spoke Architecture

Centralized GitOps Control Plane with Multi-Repo Strategic Separation

The App Repos (What)

  • • One repository per microservice/app
  • • contains: Dockerfile, src, and base Helm chart
  • • Changes trigger CI build  $\rightarrow$ image register

The Hub Repo (How)

  • • Single repo for global orchestration
  • • contains: AppProject and ApplicationSet yaml
  • • Defines the mapping logic (Generator)

The Config Repo (Where)

  • • Single repo for all environment variables
  • • contains: environment.yaml / values.yaml
  • • Allows fleet-wide updates via a single PR
